# Introduction

Welcome to the **Docusaurus Technical Book** - a demonstration of spec-driven,
technically accurate, and beginner-friendly technical writing.

## What You'll Learn

This book demonstrates how to:

- Build technical documentation using Docusaurus v3
- Follow spec-driven development principles
- Deploy automatically to GitHub Pages
- Write reproducible tutorials

## Getting Started

Get started by exploring the **[tutorial section](/docs/tutorial-basics/tutorial-intro)**.

## Project Principles

This book follows five core principles:

1. **Spec-Driven Writing** - No content without a specification
2. **Technical Accuracy** - Based on official documentation
3. **Beginner-Friendly Style** - Clear and accessible
4. **Reproducibility** - All commands and workflows are testable
5. **AI-Native Modularity** - Structured for continuous improvement

## What Makes This Different?

- Every chapter has a corresponding specification
- All commands are tested and verified
- Instructions are reproducible from a clean state
- Content is modular and maintainable
